🗄️ Guia Completo: SQL com Java (JDBC)

🔷 1. O que é JDBC?

JDBC (Java Database Connectivity) é a API que permite conectar Java com banco de dados.

Fluxo:

  • Java → Driver → Banco de Dados

🔷 2. Conexão com o Banco


package db;

import java.io.FileInputStream;
import java.io.IOException;
import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.SQLException;
import java.util.Properties;

public class DB {
    private static Connection conn = null;

    public static Connection getConnection(){
        if(conn == null){
            try {
                Properties prop = loadProperties();
                String url = prop.getProperty("dburl");
                conn = DriverManager.getConnection(url, prop);
            } catch (SQLException e) {
                throw new DbException(e.getMessage());
            }
        }
        return conn;
    }

    public static void closeConnect(){
        if(conn != null){
            try {
                conn.close();
            } catch (SQLException e) {
                throw new DbException(e.getMessage());
            }
        }
    }

    private static Properties loadProperties(){
        try(FileInputStream fs = new FileInputStream("jdbc-2\\db.properties")){
            Properties props = new Properties();
            props.load(fs);
            return props;
        } catch (IOException e) {
            throw new DbException(e.getMessage());
        }
    }
}

Explicação:

getConnection(url, user, password)

  • url → endereço do banco (jdbc:mysql://localhost:3306/db)
  • user → usuário
  • password → senha

🔷 3. Statement

Executa SQL simples


Statement st = conn.createStatement();

createStatement()

  • Cria objeto para executar SQL

🔷 4. Executando SQL

executeQuery(sql) → SELECT


ResultSet rs = st.executeQuery("SELECT * FROM produto");

executeUpdate(sql) → INSERT, UPDATE, DELETE


int rows = st.executeUpdate("INSERT INTO produto VALUES (1, 'TV', 2000)");
  • retorna quantidade de linhas afetadas

🔷 5. ResultSet

Guarda resultado do SELECT


while (rs.next()) {
    System.out.println(rs.getString("nome"));
}

Funções:

next() → vai para próxima linha

getString("coluna") → pega String

getInt("coluna") → pega int

🔷 6. PreparedStatement (IMPORTANTE)

Evita SQL Injection e melhora performance


PreparedStatement ps = conn.prepareStatement(
    "INSERT INTO produto (nome, preco) VALUES (?, ?)"
);

prepareStatement(sql)

  • ? → parâmetros

ps.setString(1, "Notebook");
ps.setDouble(2, 3000.0);

ps.executeUpdate();

setTipo(posição, valor)

  • 1 → primeiro ?
  • 2 → segundo ?

🔷 7. Transações


conn.setAutoCommit(false);

commit() → salva

rollback() → desfaz


try {
    conn.setAutoCommit(false);

    st.executeUpdate("UPDATE conta SET saldo = saldo - 100 WHERE id = 1");
    st.executeUpdate("UPDATE conta SET saldo = saldo + 100 WHERE id = 2");

    conn.commit();

} catch (SQLException e) {
    conn.rollback();
}

🔷 8. Fechamento de recursos


rs.close();
st.close();
conn.close();

Ou melhor:


try (Connection conn = DriverManager.getConnection(...);
     PreparedStatement ps = conn.prepareStatement(...)) {

}

→ fecha automaticamente

🔷 9. Exemplo Completo


import java.sql.*;

public class Main {

    public static void main(String[] args) {

        String url = "jdbc:mysql://localhost:3306/test";
        String user = "root";
        String password = "1234";

        try (Connection conn = DriverManager.getConnection(url, user, password);
             PreparedStatement ps = conn.prepareStatement("SELECT * FROM produto");
             ResultSet rs = ps.executeQuery()) {

            while (rs.next()) {
                System.out.println(
                    rs.getInt("id") + " - " +
                    rs.getString("nome") + " - " +
                    rs.getDouble("preco")
                );
            }

        } catch (SQLException e) {
            System.out.println(e.getMessage());
        }
    }
}

🔷 10. Resumo Geral

  • Connection → conecta
  • Statement → executa SQL
  • PreparedStatement → seguro (usar sempre)
  • ResultSet → resultado
  • commit/rollback → controle de transação

💡 Dica Profissional

Sempre use PreparedStatement em projetos reais.

Evita SQL Injection e melhora performance.
