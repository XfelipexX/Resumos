🔷 Tabela: Interfaces importantes em Java

Interface Descrição Exemplo Observações
Interface Define um contrato que classes devem implementar 

interface Animal {
    void fazerSom();
}
Todos os métodos são public por padrão
Comparable<T> Define ordenação natural de objetos 

class Produto implements Comparable<Produto> {

    String nome;
    double preco;

    @Override
    public int compareTo(Produto outro) {
        return Double.compare(this.preco, outro.preco);
    }
}
Usado em Collections.sort()
Comparator<T> Define ordenação externa/customizada 

Comparator<Produto> comp =
    (p1, p2) -> Double.compare(p1.preco, p2.preco);
Permite várias formas de ordenação
Iterable<T> Permite usar foreach 

for (Produto p : lista) {
    System.out.println(p);
}
Base de listas e coleções
Serializable Permite salvar objeto em arquivo 

class Produto implements Serializable {
}
Usado para persistência
Cloneable Permite clonar objetos 

protected Object clone() throws CloneNotSupportedException {
    return super.clone();
}
Uso raro hoje em dia
Runnable Representa uma tarefa executável 

Runnable r = () -> System.out.println("Rodando");
new Thread(r).start();
Usado em threads
AutoCloseable Usado em try-with-resources 

try (Scanner sc = new Scanner(System.in)) {
}
Fecha recursos automaticamente
Function<T,R> Recebe T e retorna R 

Function<Integer, Integer> f = x -> x * 2;
Programação funcional
Predicate<T> Retorna true ou false 

Predicate<Integer> p = x -> x > 10;
Filtros
Consumer<T> Executa ação sem retorno 

Consumer<String> c = s -> System.out.println(s);
Usado em forEach
