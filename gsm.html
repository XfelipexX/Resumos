hashCode e equals

São operações da classe Object utilizadas para comparar se um objeto é igual a outro

• equals: lento, resposta 100%

• hashCode: rápido, porém resposta positiva não é 100%

Equals-> Método que compara se o objeto é igual a outro, retornando true ou false.

HashCode-> Método que retorna um número inteiro representando um código gerado a partir das informações do objeto

set

chamando -> set< String> set = new HashSet, treeSet e etc<>();

HashSet - mais rápido (operações O(1) em tabela hash) e não ordenado

TreeSet - mais lento (operações O(log(n)) em árvore rubro-negra) e ordenado pelo compareTo do objeto (ou Comparator)

LinkedHashSet - velocidade intermediária e elementos na ordem em que são adicionados

Alguns métodos importantes

• add(obj), remove(obj), contains(obj)

Map< K,V>

É uma coleção de pares chave / valor

Não admite repetições do objeto chave

Os elementos são indexados pelo objeto chave (não possuem posição)

Acesso, inserção e remoção de elementos são rápidos

Uso comum: cookies, local storage, qualquer modelo chave-valor

Principais implementações:

HashMap - mais rápido (operações O(1) em tabela hash) e não ordenado

TreeMap - mais lento (operações O(log(n)) em árvore rubro-negra) e ordenado pelo compareTo do objeto (ou Comparator)

LinkedHashMap - velocidade intermediária e elementos na ordem em que são adicionados

Alguns métodos importantes

put(key, value), remove(key), containsKey(key), get(key)

put(), que associa um novo objeto valor ao objeto chave especificado;

get(), que retorna o objeto valor associado ao objeto chave especificado;

containsKey(), que indica se a chave especificada está presente na coleção;

Baseado em equals e hashCode

Se equals e hashCode não existir, é usada comparação de ponteiros
