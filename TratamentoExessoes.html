⚠️ Guia Completo: Tratamento de Exceções em Java

🔷 1. O que são Exceções?

Exceções são erros que acontecem durante a execução do programa (tempo de execução).

  • Divisão por zero
  • Acesso inválido a índice de array
  • Conversão de tipo inválida

🔷 2. Hierarquia de Exceções


Throwable
 ├── Error (erros graves do sistema)
 └── Exception
      ├── RuntimeException (não verificadas)
      └── CheckedException (verificadas)

🔷 3. Try-Catch

Usado para capturar e tratar exceções.


try {
    int x = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("Erro: divisão por zero");
}

🔷 4. Múltiplos Catch


try {
    String s = null;
    System.out.println(s.length());
} catch (NullPointerException e) {
    System.out.println("Erro de null");
} catch (Exception e) {
    System.out.println("Erro genérico");
}

🔷 5. Finally

O bloco finally sempre é executado, independente de erro. 


try {
    int x = 5 / 0;
} catch (Exception e) {
    System.out.println("Erro");
} finally {
    System.out.println("Sempre executa");
}

🔷 6. Throw

Usado para lançar uma exceção manualmente.


if (idade < 18) {
    throw new IllegalArgumentException("Menor de idade");
}

🔷 7. Throws

Indica que um método pode lançar exceções.


public void lerArquivo() throws IOException {
    // código
}

🔷 8. Checked vs Unchecked

  • Checked: precisam ser tratadas (IOException)
  • Unchecked: erros de lógica (RuntimeException)

🔷 9. Criando Exceção Personalizada


class MinhaExcecao extends Exception {
    public MinhaExcecao(String msg) {
        super(msg);
    }
}

🔷 10. Exemplo Completo


import java.util.Scanner;

public class Main {
    public static void main(String[] args) {

        Scanner sc = new Scanner(System.in);

        try {
            System.out.print("Digite um número: ");
            int x = Integer.parseInt(sc.nextLine());

            int resultado = 10 / x;
            System.out.println(resultado);

        } catch (NumberFormatException e) {
            System.out.println("Número inválido");
        } catch (ArithmeticException e) {
            System.out.println("Divisão por zero");
        } finally {
            System.out.println("Fim do programa");
        }

        sc.close();
    }
}

🔷 11. Resumo Geral

  • try: bloco que pode gerar erro
  • catch: trata o erro
  • finally: sempre executa
  • throw: lança exceção
  • throws: indica exceções

💡 Dica Final

Evite capturar exceções genéricas (Exception) sem necessidade. Sempre trate erros específicos primeiro.
